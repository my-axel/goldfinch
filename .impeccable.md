# Goldfinch Design System

## Design Context

### Users
Young professionals (25–40), tech-savvy, beginning to take retirement planning seriously. They're comfortable with data but not financial experts. The interface should make complex financial projections feel **clear and empowering** — never overwhelming or intimidating. They expect the polish and responsiveness of modern SaaS tools.

### Brand Personality
**Modern, Confident, Empowering** — like a fintech app that respects your intelligence. Data-driven but never cold. The tone is direct and assured: "here's where you stand, here's what to do."

### Emotional Goals
- **Confidence**: Users should feel in control of their financial future
- **Clarity**: Complex data presented so simply it feels obvious
- **Progress**: Every visit should reinforce forward momentum

### Aesthetic Direction
- **Visual tone**: Clean, contemporary, premium — not corporate or sterile
- **References**: Apple/iCloud (elegant simplicity, system-native feel), Stripe Dashboard (polished data presentation, generous whitespace), Mint/YNAB (clear financial visualizations, actionable insights)
- **Anti-references**: Cluttered banking apps, overly playful/gamified finance tools, dense enterprise dashboards
- **Theme**: Full dark/light mode support, system preference detection

### Design Principles

1. **Content-first clarity** — Every pixel serves comprehension. Strip decoration that doesn't aid understanding. Data should breathe with generous spacing.

2. **Confident hierarchy** — Bold use of typography weight and size to guide the eye. Primary metrics large and unmissable, supporting data secondary. No visual shouting.

3. **Progressive disclosure** — Show the essential view first, reveal complexity on demand. Scenarios, breakdowns, and details available but never forced.

4. **Purposeful color** — Purple primary for actions and emphasis. Semantic greens/yellows/reds only for status. Charts use the defined palette consistently. Never decorative color.

5. **Polished interactions** — Subtle transitions, responsive feedback, smooth state changes. The interface should feel alive but never distracting.

## Technical Design Tokens

### Colors
- **Primary**: `hsl(263.4 70% 41%)` / dark: `hsl(263.4 70% 50%)` — Purple (`#581fb2`)
- **Background**: `hsl(0 0% 97%)` / dark: `hsl(0 0% 9%)`
- **Card**: `hsl(0 0% 100%)` / dark: `hsl(0 0% 11%)`
- **Border**: `hsl(0 0% 89%)` / dark: `hsl(0 0% 15%)`
- **Chart palette**: Purple, Teal (`hsl(173 58% 45%)`), Orange (`hsl(30 80% 55%)`), Pink (`hsl(320 65% 60%)`), Red-orange (`hsl(15 75% 55%)`)
- **Status**: Green (on-track), Yellow (attention), Red (critical) — Tailwind semantic classes

### Typography
- System font stack (Tailwind defaults)
- Scale: `text-3xl` (page title) → `text-2xl` (section) → `text-lg` (card title) → `text-sm` (body) → `text-xs` (caption)
- `tracking-tight` on headings, `tabular-nums` on financial figures

### Spacing
- Page: `px-8 py-8`
- Sections: `space-y-6`, gaps `gap-4` to `gap-6`
- Cards: `p-6`, internal `space-y-2` to `space-y-3`
- Border radius: `0.5rem` base (`rounded-lg` standard, `rounded-xl` for cards)

### Components
- Custom-built on Tailwind (no component library)
- Icons: `@lucide/svelte`
- Charts: ECharts via `svelte-echarts`
- Modals: Native `<dialog>` element
- Inputs: `rounded-lg border border-border` with `focus:ring-2 focus:ring-ring`

### Accessibility
- General best practices (WCAG AA level)
- Keyboard navigation, focus visible states
- Dark mode respects `prefers-color-scheme`
- `prefers-reduced-motion` should be honored for animations
